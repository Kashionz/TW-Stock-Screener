# 個股研究浮動側邊面板設計

日期：2026-06-09

## 背景

目前研究清單為了避免桌機表格出現橫向捲軸，已調整成滿寬顯示，而個股研究面板被移到研究清單下方。這個版本雖然解決了表格可視寬度問題，但桌機閱讀流程退化成：

1. 先在研究清單中找標的。
2. 點選後仍需往下找個股面板。
3. 清單掃描與個股深讀之間的切換距離過長。

使用者明確表示「個股面板太下面了，不方便閱讀」，並在視覺比較中選擇了 `C. 全高專注面板`，也就是桌機上改成浮動側邊面板，讓閱讀重心在點選後直接切到個股研究內容。

## 目標

1. 桌機點選研究清單列後，個股面板要在可視區右側立即出現，不再要求使用者往下捲。
2. 保留研究清單作為背景上下文，讓使用者仍知道自己目前選的是哪一列。
3. 個股面板頂部要先呈現最常讀的摘要資訊，而不是直接落到長內容區。
4. 個股面板內部要以分段方式閱讀，降低單一路徑過長造成的負擔。
5. 手機與平板既有抽屜模式維持不變，不在這一輪一起重做。

## 非目標

1. 這一輪不更動篩選條件、排序、分頁或研究清單資料結構。
2. 這一輪不更動資料來源、快照格式或圖表資料計算方式。
3. 這一輪不重做手機版個股面板體驗，只維持現有抽屜模式。
4. 這一輪不新增多視窗、拖拉調整寬度或可停駐的多面板工作區。

## 選項比較

### 方案 A：輕量側滑面板

做法：

- 右側開啟較窄面板。
- 只先顯示公司摘要、少量關鍵數字與一小段內容。

優點：

- 對研究清單干擾最小。
- 切換速度快。

缺點：

- 深度研究內容仍需要額外展開或切頁。
- 無法真正解決「點進去就想直接看完整研究」的需求。

### 方案 B：研究模式側滑面板

做法：

- 右側開啟中等寬度面板。
- 內容分成摘要與多個段落，平衡清單與閱讀。

優點：

- 兼顧清單可見範圍與研究深度。
- 風險較低。

缺點：

- 閱讀沉浸感仍不如完整專注面板。
- 使用者已明確選擇更偏研究視角的方向。

### 方案 C：全高專注面板

做法：

- 桌機點列後，從右側開啟全高浮動面板。
- 面板寬度明顯大於方案 A/B。
- 面板內採「摘要優先、分段閱讀」模式。

優點：

- 最符合使用者選擇。
- 點選後能最快進入完整研究狀態。
- 保留左側研究清單作為背景上下文。

缺點：

- 會明顯降低右側表格欄位的存在感。
- 需要仔細處理面板寬度、遮罩與關閉邏輯。

## 決策

採用方案 C：全高專注面板。

理由：

- 使用者已在視覺比較中明確選定 C。
- 目前主要痛點是「個股內容太下面」，而不是清單密度不夠或摘要不夠短。
- C 能直接把研究重心拉回視窗可視區，符合桌機深讀需求。

## 體驗設計

### 桌機版閱讀節奏

桌機在研究清單點選某列後，互動節奏調整為：

1. 當前列高亮。
2. 右側滑出全高浮動面板。
3. 背景研究清單保留可見，但視覺上退到次要層。
4. 面板頂部先顯示公司名稱、代號、市櫃標記、備註摘要與關閉動作。
5. 面板中段先顯示最常讀的關鍵指標卡。
6. 面板下段再進入圖表、財報、措辭、筆記等詳細資訊。

這代表使用者在桌機上從「掃描清單」切換到「研究個股」時，不需要再進行頁面垂直位移。

### 桌機版版型

桌機（`min-width: 1181px`）使用固定浮動側邊面板：

- `position: fixed`
- `top: 24px`
- `right: 24px`
- `bottom: 24px`
- `width: clamp(340px, 36vw, 420px)`

研究清單仍維持頁面主內容，不重新回到左右分欄布局。這一輪不追求「清單與面板同等重要」，而是明確讓面板成為當前主閱讀區。

背景研究清單的處理原則：

- 保留目前滿寬表格結構。
- 面板開啟時，表格不重新流式排版。
- 使用視覺遮罩讓右側區域退到背景，避免使用者誤認為仍應主要讀表格。

### 面板資訊架構

面板內容分為四段：

1. **摘要頭部**
   - 公司名稱
   - 代號
   - 市 / 櫃標記
   - 關閉按鈕
   - 一行產業與備註摘要

2. **快速判讀卡**
   - 預設先顯示 4 個最常讀指標
   - 建議保留：
     - 月營收 YoY
     - 累計 YoY
     - 毛利率
     - EPS（季）

3. **分段導覽**
   - `總覽`
   - `圖表`
   - `財報`
   - `筆記`

4. **段落內容區**
   - `總覽`：更多關鍵數字與簡短文字摘要
   - `圖表`：近 12 個月營收圖、EPS 對比圖
   - `財報`：季度損益表與關鍵措辭檢核
   - `筆記`：備註欄位與參考連結

這個切法的目的是讓使用者先看到高度濃縮資訊，再決定要進入哪一類詳細內容，而不是一打開就面對很長的單一路徑捲動。

## 元件與程式邊界

### CSS / 結構

主要調整點位於 [index.html](/Users/kashionz/Desktop/tw-stock-screener/index.html:1)：

- `.dr`
  - 桌機恢復為固定浮動面板，而不是頁面內靜態區塊。
- `.ov`
  - 現有 overlay 保留，但桌機版改為較輕的閱讀遮罩，不採用全屏黑色遮蔽。
- `.drawer-content`
  - 需改為有清楚分段結構。
- 新增桌機用的段籤或切換控制樣式。

### UI 邏輯

主要調整點位於 [assets/app/ui.js](/Users/kashionz/Desktop/tw-stock-screener/assets/app/ui.js:1)：

- `openDrawer(code)`
  - 保持為主要入口。
  - 點不同列時應直接更新內容，不關閉再重開。
- 新增面板段落狀態，例如 `overview / charts / financials / notes`。
- 頂部摘要中的備註需使用短摘要，而不是把整段長備註完整塞進頭部。
- 詳細內容區改為依段籤切換，而不是永遠整段全部展開。

### 狀態管理

若目前 `state` 尚未有面板分段狀態，新增單一欄位即可，例如：

- `state.drawerSection = "overview"`

這一輪不需要引入新儲存模型或複雜面板狀態機。

## 互動規則

### 開啟

- 桌機點研究清單列：
  - 高亮目前列。
  - 打開浮動側邊面板。
  - 預設顯示 `總覽`。

### 切換列

- 面板已開啟時再點另一列：
  - 直接更新面板內容。
  - 保持面板開啟。
  - 一律重置回 `總覽`。

切換不同個股時，預設先回到摘要層是刻意設計，因為不同標的之間最合理的第一視角仍然是快速判讀，而不是停留在上一檔的深層段落。

### 關閉

- 點面板右上角關閉按鈕可關閉。
- 點背景遮罩可關閉。
- 若後續成本可接受，可補 `Escape` 鍵關閉，但不是必須先做。

### 響應式切換

- 桌機：使用全高浮動側邊面板。
- 平板 / 手機：延用現有抽屜模式。
- 當桌機縮到小於 `1181px`：
  - 若面板開啟中，應平順切換為現有抽屜模式。

## 錯誤與邊界情況

1. **沒有選取列**
   - 保持面板關閉。

2. **備註過長**
   - 頭部只顯示摘要版本，避免壓垮標題區。

3. **圖表或財報缺資料**
   - 段落內維持既有 fallback 文案，例如「此檔無 EPS 資料」。

4. **快速切換不同列**
   - 面板只更新當前選取列資料，不應閃爍或先關閉再開。

5. **表格仍需完整檢視**
   - 使用者可關閉面板後回到完整清單掃描模式。

## 驗證策略

### 桌機驗證

1. 開啟首頁。
2. 滾到研究清單。
3. 點任一列後：
   - 面板在右側可視區立即出現。
   - 不需要往下捲才能看到個股資訊。
   - 當前列高亮。
4. 點另一列：
   - 面板內容直接切換。
5. 切換面板分段：
   - 內容更新正常。
6. 關閉面板：
   - 背景遮罩消失。
   - 回到完整清單視角。

### 響應式驗證

1. 桌機寬度下驗證浮動側邊面板。
2. 平板寬度下驗證維持既有抽屜。
3. 手機寬度下驗證不受桌機版改動影響。

### 技術檢查

  - page identity

  - console health
  - interaction proof

## 風險與取捨

1. **面板變寬後，背景研究清單右側欄位存在感會下降**
   - 這是有意識的取捨，因為這一版明確偏向個股深讀。

2. **若面板內容仍全部直向展開，會把「太下面」問題轉移到面板內部**
   - 因此分段切換不是裝飾，而是必要條件。

3. **若頭部摘要放太多資訊，會再次擠壓可讀性**
   - 頭部只保留最關鍵元素，其餘資訊下放到總覽段。

## 實作順序建議

1. 恢復桌機固定浮動面板框架與遮罩層級。
2. 調整頭部摘要與快速判讀卡。
3. 加入分段切換與預設內容順序。
4. 驗證桌機點列、切列、關閉與斷點切換。
5. 跑既有檢查與視覺驗證。
